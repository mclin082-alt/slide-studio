Slide Studio01

Interactive HTML presentation

理想输出不是 PPT，是可演示的网页 artifact

这份 demo 把六个高价值互动模块放进同一个简单叙事里，展示你的 app 应该生成到什么水准。

点击展开信息，而不是堆满一页
拖动看前后变化，而不是两张截图
切换在同页表达多种视角和数据

按空格或右箭头翻页；右侧圆点可以跳转。

Clickable Detail02

Module 1

把复杂内容藏进可点击的详情里

适合产品截图、流程图、时间线、系统架构。观众先看到大图，再按兴趣深入。

Hotspot用户只需要描述目标

不要求用户会设计、会排版、会写代码。输入的是业务目标，系统生成的是可演示成果。

TimelineAgent 先组织故事结构

它决定页面顺序、重点、互动模块和视觉风格，而不是直接吐出静态模板。

Drilldown输出可以点击、切换、拖动

最后得到的是一个网页级 presentation artifact，可以用于 pitch、demo、作品集和技术解释。

Before / After03

Module 2

用拖动对比表达“变化”

适合讲产品改版、AI 自动化前后、运营流程优化、数据效果变化。

Before传统 deck workflow

ChatGPT 写内容、PPT 排版、Canva 找视觉、Excel 做图，最后手动拼接。

AfterAgentic artifact workflow

一次输入目标，系统组织叙事、选择模块、生成 HTML、支持演示和分享。

Progressive Reveal04

Module 3

一页内容可以按演讲节奏逐步出现

按空格或右箭头，先 reveal 当前页内容，全部出现后才翻页。

第一步：先提出问题，让观众知道为什么传统 PPT 不够用。
第二步：展示新的 artifact 工作流，把内容、设计、代码生成合成一条链路。
第三步：最后给出可点击 demo，让观众直接感受到差异。
Segment Switch05

Module 5

同一页切换不同视角

适合把一个结论从用户、产品、商业多个角度讲清楚。

用户要的是“省掉拼接工具”

他们不想在 5 个工具之间切换，只想把复杂想法变成能拿出去讲的东西。

产品要的是“模块化生成能力”

不是每次重写 HTML，而是让 agent 选择稳定模块，再由 renderer 产出可靠互动。

商业要的是“高价值窄场景”

先赢 pitch、demo、技术 proposal、AI 项目展示，而不是正面追赶 PowerPoint。

Interactive Data Toggle06

Module 6

数据不必静态铺满一页

点击不同指标，图表和结论一起切换，适合增长、ROI、竞品、用户研究。
